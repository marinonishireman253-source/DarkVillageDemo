# 室内 Cutaway 怪物设计规范

文档状态: `当前生效`
建议用途: `怪物设计、演出和压迫感方向参考`
不承担职责: `不替代玩法总纲，不展开完整战斗系统数值设计`

版本: `v1`
适用项目: `DarkVillageDemo`
适用场景: `Main`
当前推荐主怪: `梁间访客`

## 1. 设计结论

当前项目最适合的主怪，不应该是“地面上追着人跑的大型人形”，而应该是一个会利用:

- 门框
- 天花板前沿
- 房间交界处
- 局部遮挡
- 镜头过渡

来制造压力的室内 stalker 型怪物。

推荐主怪设计:

- 名称: `梁间访客`
- 类型: `室内追猎型 / 门框伏击型 / 房间过渡主怪`
- 核心卖点: `怪物永远比完整现身更可怕`

这只怪物的重点不是长时间完整露出，而是:

- 先被听见
- 再被局部看见
- 最后才短时间完整压进房间

它天然适合当前项目的 `cutaway / dollhouse / single-room framing` 视角。

## 2. 核心概念

`梁间访客` 像一具被硬塞进房屋结构里的“长手人形”。

第一眼印象:

- 像一个过高、过瘦、被折叠过的人
- 肩宽不大，但手臂极长
- 头部总是偏向一侧，像脖子已经不再按正常方式工作
- 身体不像“走进房间”，更像“从门框和吊顶里抠出来”

它不是一只冲刺型猎犬，也不是笨重巨汉。
它的恐怖来自:

- 不自然的长度
- 不完整的现身方式
- 总在房间结构边缘出现

## 3. 视觉设计

### 3.1 轮廓

轮廓必须一眼能和人区分开。

推荐轮廓:

- 窄躯干
- 异常长的前臂
- 手掌比正常人更大
- 头部小而歪
- 背部略拱

远看像:

- 一个被折断后重新拼起来的人
- 或一件挂在门梁上的旧人偶

### 3.2 比例

以玩家身高 = `1.0` 为基准:

- 站立总高: `2.4x ~ 2.8x`
- 手臂总长: `1.5x ~ 1.9x`
- 腿长: `0.85x ~ 1.05x`
- 肩宽: `0.7x ~ 0.9x`
- 手掌长度: `0.22x ~ 0.3x`

重点:

- 不要做成横向很宽的怪
- 让它“高”和“长”，不是“壮”
- 这样它在门框、天花和前景柱体附近都更有压迫感

### 3.3 材质和颜色

推荐材质方向:

- 主体: 近黑色、旧木炭色、脏灰褐色
- 皮肤暴露区域: 冷白偏蜡
- 指尖/关节: 更亮一点，方便低光里读动作
- 眼部: 最多一个弱亮点，不要做双大光眼

避免:

- 满身高饱和血红
- 大面积荧光效果
- 复杂花纹

这只怪应该首先是 `silhouette horror`，不是特效怪。

## 4. 动作设计

### 4.1 基础动作气质

动作不能流畅，必须带:

- 犹豫
- 抽动
- 关节错位感
- 突然快一下

整体节奏建议:

- 平时慢
- 靠近时突然快
- 完整现身时间短

### 4.2 推荐动画集

第一版最少做这 6 组:

1. `CeilingLurk`
   - 只露手和部分肩背
   - 像卡在门梁和天花前沿之间
2. `DoorPeek`
   - 头先歪出来
   - 一只手先抓住门框
3. `UnfoldEnter`
   - 身体从门口“展开”进房间
   - 不是正常迈步
4. `SweepAdvance`
   - 沿横向快速推进
   - 手臂先到，身体后跟
5. `FreezeObserve`
   - 完全静止盯人
   - 只做极小的头部抽动
6. `RetreatFold`
   - 倒着缩回门框或暗处

### 4.3 动作禁忌

- 禁止做成标准 humanoid run cycle
- 禁止像丧尸一样一直左右甩手冲刺
- 禁止持续完整站在房间正中间

## 5. 行为设计

### 5.1 角色定位

`梁间访客` 不是常驻战斗敌人，而是:

- 房间压迫来源
- 过渡区威胁来源
- 节奏控制器

它的任务是让玩家害怕进入下一间房，不是单纯追着玩家跑。

### 5.2 行为循环

推荐一轮标准行为:

1. `预告`
   - 顶梁轻响
   - 灯轻微摆动
   - 门框上方有影子掠过
2. `试探`
   - 一只手先出现在门口
   - 或头部在门框边缘短暂露出
3. `侵入`
   - 怪物在 1 到 2 秒内压进房间
   - 镜头保持房间 framing，不要特写切镜
4. `横向扫场`
   - 怪物占据房间局部空间
   - 强迫玩家后退、停下或等空档
5. `回收`
   - 怪物退回暗处
   - 留下声音和轻微环境变化

### 5.3 房间中的使用方式

它最适合出现在:

- 房门
- 房间与房间的交界梁
- 前景柱后
- 天花前沿下方

它不适合长期站在:

- 房间中央无遮挡区域
- 房间最前景边缘
- 背墙正中央

## 6. 房间演出规则

### 6.1 在客厅

- 从沙发后的门洞出现
- 先让吊灯动，再让手落到门框上
- 桌面和落地灯可以作为前景遮挡

### 6.2 在书房

- 先让屏幕或玻璃里出现一瞬反射感
- 再从书柜旁或门口压出
- 书房适合它“停住不动看玩家”

### 6.3 房间过渡时

这是它最强的使用点。

- 玩家刚走近门洞，不要立刻完整见怪
- 先给局部身体
- 再让镜头过渡和怪物进入几乎同时发生

这样玩家会感觉不是“切到下一间房”，而是“下一间房里已经有东西在等”

## 7. 技术实现建议

### 7.1 推荐实现形态

第一版推荐做成 `低模 3D 怪物 + 偏平光照 + 强轮廓`，不要做成单张纸片。

原因:

- 它要在门框和天花之间穿插
- 要有明显的身体厚度
- 要承受局部遮挡
- 还要在侧向镜头里读出不自然的关节

### 7.2 第一版骨架建议

最低可用骨架:

- Root
- Pelvis
- Spine
- Chest
- Neck
- Head
- Arm_L_Upper
- Arm_L_Fore
- Hand_L
- Arm_R_Upper
- Arm_R_Fore
- Hand_R
- Leg_L_Upper
- Leg_L_Lower
- Foot_L
- Leg_R_Upper
- Leg_R_Lower
- Foot_R

如果资源有限，优先保证:

- 脊柱
- 头
- 双臂

因为这只怪最主要的表演来自上半身和手。

### 7.3 碰撞和尺寸

第一版不要让碰撞跟完整肢体走。

推荐:

- 主碰撞体只包躯干
- 手臂攻击或扫击另做短时 hit volume
- 门口现身时允许部分视觉穿插，不要追求全肢体严格碰撞

## 8. 第一版最小可做版本

如果我们现在就开做，最小版本建议只做这些:

1. 一个主怪模型
2. 两组完整演出
   - `DoorPeek`
   - `UnfoldEnter`
3. 一个短推进动作
4. 一个静止盯视动作
5. 一个撤回动作
6. 两种房间使用点
   - 客厅门口
   - 书房门口

只靠这套，就已经足够支撑一段完整的“相邻房间压力测试”。

## 9. 不建议的方向

以下方向和当前项目不匹配:

- 纯地面四足爬行怪
- 体型非常巨大的 boss 型怪
- 全程完整暴露的追逐怪
- 全靠粒子和流血材质的特效怪
- 完全没有人的结构参考的抽象黑雾

这些方向不是不能做，而是不如 `梁间访客` 这么适合现在的房间切片、门框过渡和局部视角。

## 10. 可选备选方案

如果后面你想换风格，备选还有两个:

### 10.1 `楼梯寡妇`

- 更像长裙女性轮廓
- 更适合楼梯、转角、栏杆
- 优点: 氛围强
- 缺点: 在普通平层门口不如主方案好用

### 10.2 `柜中人`

- 主要从柜门、衣橱、储物间里出现
- 优点: 容易结合家具
- 缺点: 房间切换压迫感不如主方案

## 11. 最终建议

当前项目建议先只做一只主怪，不要一上来做怪物图鉴。

优先级应当是:

1. 先把 `梁间访客` 的轮廓做对
2. 再把门口出现和撤回做对
3. 最后再补攻击和更多变体

如果第一只怪就能和房间结构、镜头过渡、门框遮挡配合上，这个项目的恐怖感就会立起来。
